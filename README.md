# 보드게임 도우미 앱 - 실행 가이드

## 1. 이게 뭔가요?
서버도, 회원가입도, 인터넷 연결도 필요 없는 앱입니다.
아이패드나 안드로이드에서 브라우저로 열어서 "홈 화면에 추가"하면
아이콘을 눌러 켜는 일반 앱처럼 씁니다. 데이터는 전부 그 기기 안에만 저장됩니다.

## 2. 내 컴퓨터에서 실행해보기 (개발자 모드)

준비물: Node.js (nodejs.org 에서 무료로 설치, 버튼 몇 번이면 끝)

```bash
cd boardgame-app
npm install     # 필요한 부품들을 다운로드 (최초 1회만)
npm run dev     # 개발 서버 켜기
```

터미널에 뜨는 주소(예: http://localhost:5173)를 브라우저로 열면 앱이 보입니다.
같은 와이파이에 연결된 아이패드에서 컴퓨터의 IP 주소로 접속하면
아이패드에서도 바로 테스트할 수 있습니다.

## 3. 실제로 배포하기 (아이패드/안드로이드에서 항상 쓰기)

```bash
npm run build
```

이 명령을 실행하면 `dist` 폴더에 정적 파일(그냥 html/css/js 묶음)이 생깁니다.
이 폴더를 아래 중 아무 곳이나 무료로 올리면 끝입니다:

- Cloudflare Pages (cloudflare.com, 무료 티어)
- GitHub Pages
- Netlify

올린 뒤 아이패드 Safari에서 그 주소로 들어가서,
공유 버튼 → "홈 화면에 추가"를 누르면 진짜 앱 아이콘처럼 생깁니다.

## 4. 꼭 채워 넣어야 하는 것들

- `public/audio/` 폴더에 게임별 mp3 파일 (2분짜리, loop 재생됨)
- `public/icons/icon-192.png`, `icon-512.png` (앱 아이콘 이미지)
- `src/db.js`의 `seedGamesIfEmpty()` 안에 있는 게임 목록을
  실제 준비한 게임들로 교체

## 5. 각 부품이 뭘 하는지 (비유로 설명)

### React (화면을 그리는 도구)
레고 블록처럼 화면 조각(버튼, 목록, 입력창)을 조립해서 화면을 만드는 도구입니다.
`src/pages/` 안의 각 파일이 화면 하나씩입니다 (Home.jsx = 홈 화면).

### Dexie / IndexedDB (기기 안의 엑셀 표)
서버 없이, 이 기기 안에만 존재하는 데이터 저장 공간입니다.
`src/db.js`에 정의된 `games`, `participants`, `presets`, `sessions`가
엑셀 시트 4개라고 생각하면 됩니다. 앱을 껐다 켜도 데이터는 남아있습니다
(단, iOS Safari에서 "홈 화면에 추가"를 안 하고 오래 안 쓰면
자동 삭제될 수 있어서 꼭 홈 화면에 추가하는 걸 권장합니다).

### 서비스 워커 (오프라인 창고지기)
`public/sw.js` 파일입니다. 앱을 처음 켤 때 인터넷이 되는 상태에서
필요한 파일들(화면, BGM 등)을 미리 창고에 저장해둡니다.
다음부터는 인터넷이 없어도 창고에서 바로 꺼내 쓰기 때문에
비행기 안, 카페 와이파이 없는 곳에서도 정상 작동합니다.

### PWA manifest (앱처럼 보이게 하는 설정지)
`public/manifest.json` 파일입니다. 아이콘 모양, 앱 이름,
"홈 화면에 추가했을 때 브라우저 주소창을 숨길지" 같은 설정이 들어있습니다.

## 6. 데이터 백업 (중요)

지금 구조에서는 모든 데이터가 기기 하나에만 저장됩니다.
기기를 바꾸거나 브라우저 데이터를 지우면 캐릭터 전적이 날아갈 수 있으니,
나중에 "내보내기(JSON 파일로 저장) / 가져오기" 버튼을
Presets 화면에 추가하는 걸 다음 단계로 추천합니다.
