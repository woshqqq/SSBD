게임/캐릭터 데이터(전적, 참가자 등)는 애초에 인터넷을 안 써요. 아이패드 브라우저 안(IndexedDB)에 저장되는 거라 와이파이 유무랑 완전히 무관해요.
직접 업로드한 배경화면/BGM/캐 릭터 이미지도 마찬가지예요. 파일 선택으로 넣는 순간 그 파일 자체가 통째로 기기 안에 저장되지, 인터넷에서 매번 불러오는 게 아니에요.
앱 화면(HTML/CSS/JS)이랑 기본 이미지들은 서비스워커가 "창고"에 저장해뒀다가 오프라인일 때 거기서 꺼내줘요.
조건
최소 한 번은 와이파이/데이터 연결된 상태에서 앱을 켜서 써본 적이 있어야 해요. 한 번도 안 열어본 화면은 창고에 아직 안 담겨 있어서 오프라인일 때 안 열릴 수 있어요. 그러니까:

와이파이 있을 때 앱을 열고, 평소 쓰는 화면들(홈, 게임 상세, 게임 진행, 캐릭터)을 한 번씩 눌러서 구경
그 뒤로는 비행기 모드든 데이터 없는 곳이든 잘 열림
하나 더 챙길 것
꼭 "홈 화면에 추가"로 설치해서 쓰세요. Safari 탭으로만 열어서 쓰면 iOS가 "7일 넘게 안 쓴 사이트"의 저장 데이터를 자동으로 지워버릴 수 있어요. 홈 화면에 추가된 앱(아이콘으로 여는 것)은 이 자동삭제 대상에서 빠져서 훨씬 안전해요.

정리하면: 한 번 온라인으로 훑어보고 → 홈 화면에 추가해서 쓰면 → 이후로는 와이파이 없이도 완전히 정상 작동합니다.

1. 실행하기

npm run dev
브라우저에서 http://localhost:5173 접속 (지금도 백그라운드에서 켜져 있을 수 있어요). 실제 배포할 땐 npm run build 후 dist/ 폴더를 정적 호스팅하면 됩니다.

2. 홈 화면 (게임 선택)
검색창에 이름 입력해서 게임 필터링
"+ 새 게임 만들기": 누구나 가능. 이름/장르/비고(인원 등)/서포트 도구 타입만 정하면 바로 생성
하단 "관리자" 버튼: 처음 누르면 비밀번호를 새로 정하는 창이 뜸(이후엔 그 비밀번호를 입력해야 켜짐). 켜지면 각 게임 카드에 "수정" 버튼이 생겨서 이름/장르/비고/BGM을 바꿀 수 있어요.
3. 게임 상세 화면 (게임 하나 클릭했을 때)
배경화면 카드: 파일 선택하면 바로 교체됨 (관리자 아니어도 가능)
배경음악 카드: 파일명만 보여줌 (재생은 게임 시작 후 자동)
참가자 등록: 세 가지 방법
프리셋 불러오기 — 이미 등록된 캐릭터 선택
참가자 생성 — 이름만 입력해서 캐릭터를 새로 만들며 바로 참가
게스트 생성 — 이름만 입력, DB에 저장 안 되는 1회용 참가자
참가자 1명 이상이어야 "게임 시작" 버튼이 눌림
(자원 카운터 타입 게임이면) **"리소스 관리"**에서 카운터 개수(0~10)와 라벨을 미리 설정 가능
4. 게임 진행 화면
좌상단: 경과시간 + 음소거 버튼
하단: (리소스 카운터가 있으면) 리소스 바 → 참가자 이름 칩 → 시계 아이콘 + "게임 종료"
참가자 이름 클릭 → 팝업에서 "수정"(다른 캐릭터로 교체) / "승리"(+1, 이펙트 표시)
시계 아이콘 → 스톱워치 또는 타이머(3/5/10/15분 또는 직접 입력) 선택. 팝업을 닫아도 타이머는 계속 흐르고, 아이콘 자체가 남은/경과 시간을 보여줌. 타이머가 끝나면 진동+알림음
게임 종료 → "승리자 체크할까요?" 확인 → 예 선택 시 체크박스로 승리자 여러 명 한번에 선택 가능 → 완료하면 결과 화면으로
5. 결과 화면
게임 정보, 우승자(동점이면 전부), 플레이 시간, 참석자별 승리 횟수. "게임 선택창으로"로 홈 복귀.

6. 캐릭터 화면
상단 내보내기/가져오기로 전체 데이터 백업·복원
새 캐릭터 등록: 이름/직업/특기/취미 + 이미지(안 고르면 기본 이미지 9종 중 랜덤 배정)
각 카드의 수정은 비밀번호 없이 바로 열림, 삭제는 확인창만
7. 나중에 직접 바꿔 넣을 것들
public/characters/*.png — 지금은 심플 실루엣, 같은 파일명으로 실제 일러스트 교체
public/audio/timer-end.wav — 지금은 비프음 3번, 원하는 알림음으로 교체
public/backgrounds/background_default.png — 게임에 배경화면을 안 정했을 때 쓰



아이패드에서 쓰기
아이패드 Safari로 위 주소 접속
공유 버튼 → "홈 화면에 추가"
생긴 아이콘으로 열면 끝
앞으로 업데이트할 때
코드를 더 고치고 나면 이제부터는:


git add .
git commit -m "설명"
git push
이 세 줄만 하면 자동으로 다시 빌드+배포됩니다 (Actions 탭에서 진행 상황 확인 가능). 지난번에 말씀드린 대로, public/sw.js의 CACHE_NAME 버전 문자열은 업데이트할 때마다 하나씩 올려주셔야 아이패드에서도 최신 버전을 받아가요.

public/sw.js(서비스워커)는 "한 번 창고에 넣어둔 파일은 인터넷 확인 안 하고 그냥 창고 것부터 꺼내준다"는 규칙으로 동작해요. 그래서 이런 문제가 생겨요:

아이패드에서 앱을 처음 열면 → 그 순간의 파일들을 통째로 "창고"(캐시)에 저장해둠
나중에 제가 코드를 고쳐서 GitHub에 새로 배포함
아이패드에서 다시 앱을 열어도 → 창고에 이미 있으니까 인터넷 확인도 안 하고 옛날 버전을 그대로 보여줌 ← 이게 문제
왜 CACHE_NAME 줄을 바꿔야 업데이트가 되냐면
브라우저는 "sw.js 파일 내용이 예전이랑 한 글자라도 다르면" → "아, 창고지기가 바뀌었네" → 창고를 통째로 비우고 다시 채움, 이라는 규칙으로 동작해요.

그런데 sw.js 파일 자체는 저희가 거의 안 건드려요. React 코드(화면, 버튼 등)를 아무리 많이 고쳐도 sw.js는 그대로라서, 브라우저는 "creativity창고지기가 안 바뀌었네 → 그대로 옛날 창고 써야지"라고 착각해버려요.

그래서 일부러 sw.js 안에 있는 버전 숫자 하나만 매번 바꿔서, "어 창고지기 옷이 바뀌었네" 하고 브라우저가 눈치채게 만드는 거예요.

실제로 뭘 하면 되냐면
public/sw.js 파일 5번째 줄이 이렇게 생겼어요:


const CACHE_NAME = 'boardgame-cache-v1';
앱을 새로 배포할 때마다 이 줄의 숫자만 하나씩 올리면 됩니다:


const CACHE_NAME = 'boardgame-cache-v2';   // 다음 업데이트 때
const CACHE_NAME = 'boardgame-cache-v3';   // 그다음 업데이트 때
그게 다예요. 이 한 줄 안 바꾸면: 코드는 GitHub에 새로 올라가 있는데 아이패드는 옛날 화면만 계속 보여줌. 이 한 줄만 바꾸면: 아이패드에서 앱 껐다 켤 때 자동으로 최신 버전으로 갈아탐.

참고로 — 지금부터는 제가 코드를 고칠 때마다 이 숫자도 알아서 같이 올려드릴게요. 사장님이 직접 신경 쓰실 일은 없고, 그냥 "이런 이유로 이 파일이 같이 바뀌는구나" 정도만 알고 계시면 됩니다.
